# 中草药溯源系统（TCM Traceability System）— RuoYi-Vue MVP

本项目基于 **RuoYi-Vue（Spring Boot + Vue2 + Element UI）** 实现一个可运行的「中草药溯源」MVP：支持企业端建档、批次管理、环节追加记录与批次发布生成溯源码；支持消费者溯源查询与评价，实现“**一码全链路**”的可视化展示。

## 1. 现在系统可以干嘛

### 1.1 企业端（生产/经营方）
- **工作台（Dashboard）**：展示基地/产品/批次数量统计、快捷入口与溯源流程设计对照。
- **种植基地管理**：基地建档（省市区、详细地址、面积、报告链接等）。
- **中草药产品档案**：产品编码、名称、产地、库存等信息维护。
- **生产批次管理**：选择基地 + 产品创建批次；支持“发布”生成溯源码。
- **种植加工环节记录（只增不改）**：按批次追加环节数据（种子/种苗、种植、灌溉、施肥、除草、采收、初加工、包装）。
- **发布溯源码**：为批次生成唯一 **UUID32** 溯源码并生成二维码图片（当前为可展示的 base64 图片，后续可替换为可扫码二维码）。

### 1.2 消费者端
- **溯源查询**：输入溯源码查询并可视化展示：
  - 批次/产品/基地信息
  - **环节步骤条**（按环节是否登记高亮）
  - **环节时间线**
  - 二维码展示与“复制溯源码”
  - 评价均分与评价条数汇总
- **消费者评价**：对批次进行评分与评价。

### 1.3 权限与角色（RBAC）
- **企业角色**：`tcm_enterprise`
- **消费者角色**：`tcm_buyer`
- **公共接口**：提供匿名溯源查询接口，支持“扫码即查”场景。

## 2. 角色能做什么（详细说明）

> 说明：系统基于 RuoYi 的 RBAC 权限模型。下面按“角色 → 典型任务”描述当前 MVP 已实现的能力。

### 2.1 企业端角色：`tcm_enterprise`
- **目标**：完成“基地建档 → 产品建档 → 批次建档 → 环节记录 → 发布溯源码”的闭环，并对外提供可查的溯源数据。
- **可执行操作**
  - **工作台**
    - 查看基地/产品/批次数量统计
    - 从快捷入口进入各业务页面
  - **基地管理**
    - 新增/编辑/删除基地档案（地址、面积、报告链接等）
    - 作为“批次”的上游基础数据
  - **产品档案**
    - 新增/编辑/删除产品信息（编码、名称、产地、库存等）
    - 作为“批次”的上游基础数据
  - **批次管理**
    - 新增批次（必须绑定 1 个基地 + 1 个产品）
    - 编辑/删除批次（未发布阶段）
    - **发布批次**：生成唯一溯源码（UUID32）与二维码图片，并进入“可对外查询”状态
  - **环节记录（只增不改）**
    - 为指定批次追加环节记录：种子/种苗、种植、灌溉、施肥、除草、采收、初加工、包装
    - 每条记录包含：环节类型、内容、时间、操作人
  - **溯源码查询（企业自查/验真）**
    - 输入溯源码查看对外展示效果（与消费者看到的一致）

### 2.2 消费者角色：`tcm_buyer`
- **目标**：通过“输入/扫码溯源码”验证真伪与查看全链路信息，并对产品与企业进行评价反馈。
- **可执行操作**
  - **溯源查询**
    - 输入溯源码查询（或扫码后跳转查询）
    - 查看批次、产品、基地信息
    - 查看环节步骤条与时间线（判断是否有缺失环节）
    - 查看消费者评价汇总
  - **评价管理**
    - 对批次提交评价：产品评分、企业评分、评价内容
    - 查看自己或全体的评价记录（取决于权限菜单配置）

### 2.3 系统管理员：`admin`（RuoYi 默认能力）
- **目标**：进行系统级管理（用户、角色、菜单、字典、参数等）。
- **典型操作**
  - 在【系统管理】中分配 `tcm_enterprise` / `tcm_buyer` 角色给用户
  - 在【菜单管理】中维护 TCM 菜单（如工作台、基地、产品、批次、环节、溯源查询、评价）
  - 在【参数设置】中配置验证码开关等（演示环境可关闭，生产建议开启）

## 3. 界面能做什么（页面说明）

> 前端页面主要在 `ruoyi-ui/src/views/tcm/` 下。下面按页面逐一说明“你能在这个界面完成什么”。

### 3.1 工作台：`/tcm/dashboard`
- **用途**：企业端入口页，展示业务概览与流程对照。
- **主要内容**
  - 统计卡片：基地数、产品数、批次数
  - 快捷入口：基地、产品、批次、环节、溯源查询、评价
  - 标准溯源流程 Steps：用于软件设计对照展示（从种苗到包装）

### 3.2 种植基地管理：`/tcm/base`
- **用途**：维护基地档案，为批次提供上游数据。
- **能做什么**
  - 查询/新增/编辑/删除基地
  - 维护字段示例：基地名称、省市区、详细地址、面积、土壤/水质报告链接、备注等

### 3.3 中草药产品档案：`/tcm/product`
- **用途**：维护产品档案，为批次提供上游数据。
- **能做什么**
  - 查询/新增/编辑/删除产品
  - 维护字段示例：产品编码、产品名称、产地、库存、备注等

### 3.4 生产批次管理：`/tcm/batch`
- **用途**：以“基地 + 产品”为归属创建批次，并发布生成溯源码。
- **能做什么**
  - 查询/新增/编辑/删除批次
  - 批次创建时选择：基地、产品；填写批次号、播种/采收日期（可为空）等
  - **发布批次**
    - 生成溯源码（UUID32）
    - 生成二维码图片（用于查询页展示/扫码入口）
    - 发布后可被公共接口查询

### 3.5 种植加工环节：`/tcm/process`
- **用途**：为批次追加“环节记录”，形成可追溯时间线。
- **能做什么**
  - 选择批次、选择环节类型、填写环节内容与时间、操作人
  - 新增后不可修改（只增不改）——保证追溯链条的可信性

### 3.6 溯源码查询：`/tcm/trace`
- **用途**：企业自查/消费者查询的统一入口页（也可对接扫码跳转）。
- **能做什么**
  - 输入 32 位溯源码进行查询
  - 展示二维码与溯源码，支持复制
  - 展示“批次/产品/基地”信息
  - **环节步骤条**：按环节是否登记展示“已登记/未登记”
  - **环节时间线**：按时间展示每条环节记录
  - **评价汇总**：产品均分、企业均分、评价条数

### 3.7 评价管理：`/tcm/review`
- **用途**：消费者对批次进行评价，形成口碑反馈。
- **能做什么**
  - 新增评价：产品评分、企业评分、评价内容
  - 查看评价列表（用于管理/审核的基础能力，后续可扩展为敏感词/审核流）

## 4. 技术栈
- **后端**：Spring Boot（RuoYi-Vue）、Spring Security、Redis、MySQL
- **前端**：Vue 2、Vue Router、Element UI

## 5. 数据库与初始化脚本

脚本位于 `sql/`，建议用 UTF-8 编码执行。

- **`sql/tcm_mvp.sql`**
  - 创建 `biz_tcm_*` 溯源业务表：基地/产品/批次/环节/溯源码/评价
  - 初始化角色、菜单、权限绑定（含 TCM 模块与工作台菜单）
- **`sql/tcm_test_users.sql`**
  - 初始化测试账号（仅测试用途，勿用于生产）
- **`sql/tcm_dashboard_menu.sql`**
  - 给已存在的库补充“工作台”菜单（如果早期执行过旧脚本）

## 6. 本地启动（Windows）

你当前使用的步骤（见 `启动.txt`）可直接运行：

1. 启动 MySQL
2. 启动 Redis
3. 前端：
   - `cd ruoyi-ui`
   - `npm run dev`
4. 后端打包：
   - `mvn clean package -DskipTests -Dexec.skip=true`
5. 后端启动：
   - `java -jar ruoyi-admin/target/ruoyi-admin.jar`

## 7. 测试账号（执行 `sql/tcm_test_users.sql` 后）

## 8. 快速联调（可选）

提供链路联调脚本（新增基地→新增产品→新增批次→新增环节→发布溯源码→匿名查询→买家评价）：

- `scripts/tcm_smoke/smoke_tcm.ps1`

为避免在开源仓库硬编码口令，该脚本通过环境变量读取密码：

```powershell
$env:TCM_ENT_PWD="admin123"
$env:TCM_BUYER_PWD="admin123"
```

然后运行脚本即可。

## 9. 开源安全说明
- 后端配置支持用环境变量覆盖 `DB_URL/DB_USERNAME/DB_PASSWORD`、`TOKEN_SECRET`、`DRUID_PASSWORD` 等敏感字段。
- 请不要把生产环境真实密码/密钥提交到仓库。
